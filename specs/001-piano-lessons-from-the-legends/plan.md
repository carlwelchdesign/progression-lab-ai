# Implementation Plan: Piano Lessons from the Legends

**Branch**: `001-piano-lessons-from-the-legends` | **Date**: 2026-04-13 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `/specs/001-piano-lessons-from-the-legends/spec.md`

## Summary

Add a "Piano Lessons from the Legends" feature where authenticated users choose a legendary pianist and receive a 3-lesson AI-generated curriculum personalized to their skill level. Each lesson follows a strict 6-part narrative arc (Hook → Concept → Hear+See It → Try It → Why It Works → Real Song Connection) designed for retention, not just information transfer. Curricula are cached per user/musician pair and invalidated automatically when the musician's prompt template is updated.

A prior prototype exists on `spike/explore-claude`. That prototype proved the AI generation and data model are sound, but the lesson UX was too text-heavy and didn't feel like the musician was actually teaching. This implementation starts from scratch on top of the proven DB schema.

## Technical Context

**Language/Version**: TypeScript 5.8 / Node.js 20+  
**Primary Dependencies**: Next.js 15 App Router, React 19, MUI v7, Prisma 5, OpenAI SDK (Responses API), `piano-chart` (interactive keyboard)  
**Storage**: PostgreSQL via Prisma — new models: `MusicianProfile`, `GeneratedCurriculum`, `LessonProgress`  
**Testing**: Jest (unit/integration), Playwright (e2e)  
**Target Platform**: Web (server-rendered + client interaction)  
**Project Type**: Web application feature within existing Next.js monorepo  
**Performance Goals**: Curriculum generation ≤ 10s P90; lesson UI interaction < 100ms  
**Constraints**: AI generation counts against existing `aiGenerationsPerMonth` quota; no tier gate; graceful MIDI fallback  
**Scale/Scope**: ~4 musicians at launch; ~3 lessons per musician; one curriculum per user/musician pair

### SOLID Constraints (enforced across all implementation)

- **SRP**: Route handlers validate + delegate only. Business logic (curriculum generation, skill assessment, profile generation) lives in dedicated service functions under `features/musician-styles/services/` and `features/lessons/services/`.
- **OCP**: New exercise types, lesson step formats, and musician profile sources are added by extending `GeneratedStep` union types and handler registries — not by adding `if/else` branches to existing renderers.
- **LSP**: `MidiService` interface must be fully implementable by both `WebMidiService` (production) and `MockMidiService` (tests). No partial implementations.
- **ISP**: Components receive the narrowest possible prop types. `LessonStepText` receives only `TextStep`, not the full `GeneratedLesson`. `MusicianCard` receives only `MusicianProfileSummary`, not the full DB model.
- **DIP**: API route handlers receive service functions as parameters (or via factory). No direct `import { openai }` inside route files — AI calls are encapsulated in `generateCurriculumBatch(musician, user, options)` and `generateMusicianProfile(name, genre)` service functions.

## Implementation Checklist: Low-Click Flow

### Phase A: Flow Controller

- Create a dedicated flow controller in `LessonPlayer` with deterministic step transition states.
- Ensure one reason to change: step progression logic lives in controller/hook, not UI button components.

### Phase B: Auto-Advance Behavior

- Text steps: single `Continue` action + keyboard shortcut parity.
- Exercise steps: auto-complete on MIDI `targetNotes` match and auto-advance after 700-1200 ms.
- Add duplicate-event guards for held/sustained MIDI notes.

### Phase C: API + Persistence

- Include `autoAdvance` in progress writes for each step completion.
- Return `nextStepIndex` from progress API for smoother client transitions.
- Preserve progress on route changes and page refresh.

### Phase D: Observability + QA

- Emit metrics: `manualClicks`, `autoAdvanceCount`, `backtrackCount`.
- Validate target: mean manual clicks per completed step <= 1.2.
- Add unit tests for transition state machine and debounce guards.
- Add Playwright scenario for keyboard progression + MIDI auto-advance.
